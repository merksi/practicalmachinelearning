Introduction

This report describes the result of the project for the coursera project “practical machine learning”. The description of the project can be found here: https://class.coursera.org/predmachlearn-030/human_grading/view/courses/975199/assessments/4/submissions

The data for this project has come from this source: http://groupware.les.inf.puc-rio.br/har.

Using devices such as Jawbone Up, Nike FuelBand, and Fitbit it is now possible to collect a large amount of data about personal activity relatively inexpensively. In this project, data will be used from accelerometers on the belt, forearm, arm, and dumbell of 6 participants whowere asked to perform barbell lifts correctly and incorrectly in 5 different ways.

The goal of the project is to predict the manner in which they did the exercise. This is the “classe” variable in the training set. Any of the other variables might be used for the prediction.

The report should contain the following elements - describing how you built your model, - how you used cross validation, - what you think the expected out of sample error is, and - why you made the choices you did.

Finally, the evental prediction model will be used to predict 20 different test cases.

Machine learning algorithm including cross-validation

This classification problem requires a machine learning algorithm that can predict multiple classes and that can handle non-linear data.

Therefore the choice has been made to use the random forest algorithm which is often very accurate.

A con of the random forest algorithm is the risk of overfitting. Therefore cross-validation is applied to asses the accuracy

# set the seed
set.seed(1235)
# do training on only part of the data to limit calculation time
smallTrain<-createDataPartition(y=trainDataSub$classe,p=0.5,list=FALSE)
trainDataSubSmall<-trainDataSub[smallTrain,]
# fit the model using Random Forest and Cross validation 
modelFit<-train(as.factor(classe)~.,data=trainDataSubSmall, method="rf", 
                trControl=trainControl(method="cv",number=5),
                prox=TRUE,allowParallel=TRUE)

Expected out-of sample error.

From the final model, the estimate of the out of sample error is 8.56%. Since the training was done on a subset of the data, it is possible to confirm this estimate with the remainder of the training data.

# do training on only part of the data to limit calculation time
crossDataSubSmall<-trainDataSub[-smallTrain,]
# fit the model using Random Forest and Cross validation 
crossPred<-predict(modelFit,crossDataSubSmall)
